添加 RequireJS § 2

注意: 针对于 jQuery 的建议，见 如何与 jQuery 集成页面

首先我们假设你会把所有的 JavaScript 文件都放在项目的 "scripts" 目录中。现在假设你有一个项目，项目中有一个 project.html 文件，还有一些脚本文件，那么此时的目录结构应该是这个样子：

  • project-directory/
    • project.html
    • scripts/
      • main.js
      • helper/
        • util.js

添加 require.js 到 scripts 目录，它看起来应该是这样的：

  • project-directory/
    • project.html
    • scripts/
      • main.js
      • require.js
      • helper/
        • util.js

若要充分发挥优化工具的作用，建议把 HTML 中内联的脚本都移除出去，仅引用 require.js 并使用 require 来加载脚本：

<!DOCTYPE html>
    <html>
        <head>
            <title>My Sample Project</title>
            <!-- data-main attribute tells require.js to load
                 scripts/main.js after require.js loads. -->
            <script data-main="scripts/main" src="scripts/require.js"></script>
        </head>
        <body>
            <h1>My Sample Project</h1>
        </body>
    </html>

在 main.js 中，可以使用 require() 来加载其他需要的脚本。这保证了单一的入口，因为 data-main 指定的脚本是异步加载的。

require(["helper/util"], function(util) {
        // 这个函数会在 scripts/helper/util.js 加载完成后调用。
        // 如果 util.js 调用 define(), 那么这个函数会在
        // util 的依赖（脚本）加载完成后调用，
        // util 参数会一直保持着模块 "helper/util" 的值。
    });

这样就可以加载 helper/util.js 脚本了。要充分使用 RequireJS 的能力， 查看 API 文档了解更多关于定义与使用模块的信息。
